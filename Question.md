# 想定される質問一覧
## ・面接でのテクニック
### 1. 「はい/いいえ」で答えられる質問に対しては、まず「はい」か「いいえ」で答える。ダメな例の典型が、「はい」か「いいえ」を答えずに長々と説明を始めるパターン。
### 2. 質問への回答は端的かつ簡潔に。質問の意図がよくわからない場合は、「……という意味ですか？」と質問意図を聞き返して確認する。
### 3. 研究の根本に関わる部分や前提・仮定の部分以外で、わざと説明を省略しておき、質問するように仕向ける、という高等テクニックもある。初心者にはお薦めしない。

| それ説明できる | FCNは畳み込み層のみで構成されるネットワークで、最終層を1*1の畳み込み層としているネットワーク．・FPNは畳み込みによって得られた特徴マップをピラミッド型に伝播させていく手法 |
| なぜ他の手法ではなく,Mask R-CNNにしたの？ | 人間の身体のような複雑に入り組んだ画像をセグメーテンションするのに優れた手法だと考えたからです |
| なぜself attention | 入力ベクトル内でのattentionスコアを求めることができ, <br>画像と文章のどこに注目しているのかを可視化できるから|

| attentionはなぜ良い結果が出るの？ | 文章や画像の特定の部分に注意を向けるよう学習していくので, より人間に近い形で画像や文章を認識することができるから |
| 単語の埋め込み表現とCNNによって得られた特徴量はどうやって組み合わせるの(マルチモーダル特徴量) | ある位置pの画像の特徴マップをvpとして、n番目の単語埋め込み表現をenした時、それぞれのベクトルをconcatし、それを全ての位置において行うことで求められる |
| マルチモーダル特徴量からself attentionをどうやって求めるの？ | マルチモーダル特徴量からクエリ・キー・バリューを生成し、クエリとキーの内積を求め(マルチモーダル特徴量の次元の平方根でスケーリングしてあげるといいかも)softmaxで確率に落とし込む、その後バリューとの内積を計算することで求まる。 |
| attentionがよい結果を返すためにはどうする？ | マルチモーダル特徴量からQKVを生成するときに何回か線形投影して、multi head attentionのようにする |
| ROIpoolingの欠点は | 特徴マップからROIを抽出する際の丸め誤差が大きい |
| ROIAlignのいいところは | 特徴マップからROIを抽出する際のサブピクセルの誤差を最小限に抑えることができる |
| region propsal networkにテキストを使わないのは何故か | RPNは画像の中の物体がどこに写っているかとその矩形を出力するものであるので、言語表現は必要ないと考えたから |

| U-netの派生手法は他に知ってる | R2U-net(U-netにresidual構造とrecurrent構造を導入したモデル：畳み込みを再帰的に行い，パラメータを増やすことなく何度も畳み込みを実行できる．)・3D U-net(U-netを３次元に拡張したモデルで3次元畳み込みを用いることにより，MRIなどの３次元画像にも対応できるようになった．) |
| U-net++を使わないのはなぜ | U-net++はセマンティックセグメンテーションであり，インスタンスごとにセグメンテーションできないから． |
